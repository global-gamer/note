# Phase 6 – Stabilize & Automate Ops

**Ziele:** Prozesse dokumentieren, Workload reduzieren, Qualität sichern bevor Wachstumssprints starten.

## Key Workstreams
- SOPs für Moderation, Events, Support in interner Knowledge Base (Notion/Confluence) ablegen.
- Automatisierte Reports: Discord KPIs, Forum Stats, Financial Dashboard.
- Incident Response Playbooks + On-Call Rotation für Ops Team.
- Node02 final deployen (oder Puffer-VM) + Backups/Monitoring clonen.

## Economic Focus
- Kostenkontrolle: Verträge renegotiaten (Server, SaaS) für Mengenrabatte.
- Break-even-Point kalkulieren; Ziel: 60–70 % Kosten durch Supporter decken.

## Check / KPIs
- Jede Kernrolle hat dokumentierte SOP + Vertretung.
- Ops Alerts <5/Monat, Mean Time To Recovery <30 Min.
- Supporter Revenue deckt ≥50 % der monatlichen Infra-Kosten.

## Milestones & Owners
| Deliverable | Owner |
| --- | --- |
| SOP Library (Moderation, Events, Support) fertig | Community Host + Player Support Lead + Event Producer |
| Automated KPI Reports (Discord, Forum, Finance) | Data & Insights Analyst |
| Incident Response Playbook + On-Call Schedule | Ops Captain |
| Node02 Deployment (if required) + Backup/Monitoring Clone | Server Pilot |
| Supporter Revenue Coverage ≥50 % | Growth Captain + Supporter Lead |

## Notes
- Jedes SOP-Dokument referenziert Templates & Tools in Vault/Notion.
- Automations (Zapier, APIs) dokumentieren → Onboarding neuer Team-Mitglieder.
